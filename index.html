How many types of database are there?

Relational database

  • A relational database is a database based on the relational model of data.
  • RDBMS - relational database management system
  • SQL - Structured Query Language

Table as a basis

  • Row (tuple, record) - a data set representing a single item
  • Column (attribute, field) - a labeled element of a tuple, e.g. "Name" or "Price"
  • Table (relation, base relvar) - set of columns and rows
  • View (result set, derived relvar) - a set of tuples; a data report from the RDBMS in response to a query

Relations between tables

  • Primary key uniquely identifies each record in a table. Primary keys must contain UNIQUE values, and cannot contain NULL values.
  • Foreign key is a field (or collection of fields) in one table, that refers to the PRIMARY KEY in another table. The table with the foreign key is called the child table, and the table with the primary key is called the referenced or parent table.
  • Composite key is a primary key composed of multiple columns used to identify a record uniquely

Relations between tables

  • One to one: For every record in the parent table, there is one and only one record in the child table.
  • One to many: For every record in the parent table, there are one or more related records in the child table.
  • Many to many: For every record in the parent table, there are many related records in the child table, and for every record in the child table, there are many related records in the parent table.

Normal forms

  • Database normalization is the process of organizing the attributes of the database to reduce or eliminate data redundancy (having the same data but at different places).
  • Most database systems are normalized database up to the third normal forms in DBMS.

Normal forms

  • First normal form - relation should not contain composite or multi-valued attribute.

Normal forms

  • Second normal form - 1NF + relation must not contain any partial dependency.

Normal forms

  • Third normal form - 2NF + relation must not contain any transitive dependency.

Normal forms

  • If no database table instance contains two or more, independent and multivalued data describing the relevant entity, then it is in 4th Normal Form.
  • A table is in 5th Normal Form only if it is in 4NF and it cannot be decomposed into any number of smaller tables without loss of data.
  • 6th Normal Form is not standardized, yet however, it is being discussed by database experts for some time.
